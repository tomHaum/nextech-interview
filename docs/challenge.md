## Challenge Overview

Using the Hacker News API, create a solution that allows users to view the newest stories from the feed.

Your solution should include an Angular front-end application that communicates with a C# .NET Core RESTful API.

Hacker News API:
https://github.com/HackerNews/API

## Front-End Requirements (Angular)

Your front-end UI should include:

- A list displaying the newest stories
- Each story should include:
  - The title
  - A link to the news article (note that some stories may not include a hyperlink)
- A search feature that allows users to filter or find stories
- A paging mechanism (displaying 200 stories on a single page is a bit much!)
- Automated tests for your code

## Back-End Requirements (C# .NET Core API)

Your back-end API should include:

- Use of dependency injection (since it’s built in, we encourage you to leverage it)
- Caching of the newest stories
- Automated tests for your code

While an Angular-only solution could work for this scenario, we specifically want to see your C# and .NET Core experience in action.

## Agentic AI Tool Usage (Required)

As part of this coding challenge, the use of AI-assisted development tools is required.

Modern software development increasingly involves engineers working in partnership with AI systems. This exercise is designed to evaluate how effectively you can collaborate with AI as part of your workflow.

### What This Means

You are expected to actively use one or more AI tools (e.g., ChatGPT, Claude, GitHub Copilot, Cursor, Copilot Chat, etc.) while completing this challenge.

AI usage should be a deliberate and visible part of your process, including:

### Planning & Design

- Breaking down the problem and clarifying requirements
- Designing front-end and back-end architecture

### Implementation

- Generating or accelerating boilerplate code
- Writing Angular components, services, and tests
- Writing C#/.NET Core controllers, services, and tests

### Debugging & Iteration

- Investigating errors or failed tests

### Testing & Quality

- Generating test cases
- Validating edge cases (e.g., missing story URLs, paging behavior)
